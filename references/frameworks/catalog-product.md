# Catalog Product — Multi-SKU Grid + Lookbook Framework

## Purpose & When To Use This Framework

Use this framework when the operator needs a print-ready or PDF-distributed product catalog covering multiple SKUs — typically 20+ products across categories. The audience is a buyer, procurement specialist, retail merchandiser, or end-consumer browsing the product line. The catalog must answer: "What products do you sell, what are their specs, what do they cost, and how do I order?"

Per the framework-structures-2026 research backbone (§5 Product Catalog framework), catalogs use technical grid systems to manage SKU density while incorporating "Lookbook" editorial elements. The 4-column system is the technical standard for spec tables and high-density indexes; the 3-column system is reserved for lookbook lifestyle photography that humanizes SKUs and provides editorial breathing room.

Default page count is 16 (cover + TOC + 12 product grid pages + pricing + order CTA + contact). Compress to 8 for boutique single-category catalog. Expand to 24+ for full-line multi-category catalog. Above 24 pages, consider sectioning with chapter dividers.

## Narrative Arc / Page Sequence

1. **Page 1 — Cover.** Brand name + catalog edition (e.g., "Spring 2026" or "AW26 Collection") + signature visual. The cover visual is one hero SKU or one editorial location image, not a product collage.
2. **Page 2 — Table of Contents / Index.** Navigational map. Categories listed with page numbers. For 16+ page catalogs, TOC is mandatory per the research §5 do-not pattern (navigational failure is a dominant catalog failure).
3. **Pages 3–N — Product Grid Pages.** Each page presents 3–4 SKUs per row, 2–3 rows per page (so 6–12 SKUs per page). Per SKU block: product photo (consistent treatment across catalog), SKU code, product name, spec table (3–6 lines: dimensions, material, weight, capacity, color options), price or "request quote." 4-column system for technical/dense; 3-column system for lookbook editorial.
4. **Optional Lookbook Spreads (interspersed every 4–6 pages).** Editorial spreads using 3-column lifestyle photography to humanize SKUs. Per the research §5: lookbook breaks up grid density and provides "editorial breathing room." One lookbook page per 5–8 grid pages.
5. **Page N+1 — Pricing Reference.** Consolidated pricing table (if not embedded per-SKU) or pricing tier explanation (volume discount, dealer pricing, MOQ thresholds). For wholesale catalogs, this page bears "Harga MOQ" or "Distributor pricing on request" framing.
6. **Page N+2 — Order CTA.** Order channel: distributor list, order form download QR, e-commerce link, or sales contact. Single CTA per order channel. Order protocol clearly stated (MOQ, lead time, shipping zones).
7. **Final Page — Contact.** Brand HQ address, distributor inquiry contact, customer service contact, regulatory disclosure (BPOM, SNI, Halal if applicable for Indonesian market).

## 7-Step Content Checklist

1. **TOC mandatory for 8+ page catalogs.** Per the research §5, navigational failure is a dominant catalog do-not pattern. TOC lists category + start page. Optional sub-index by SKU for 16+ page catalogs.
2. **Grid consistency across all product pages.** Same column system (4-col or 3-col), same SKU block height, same photo treatment, same spec table row count. Inconsistent grid signals amateur production.
3. **Product photography consistent treatment.** Either all white-background studio, all lifestyle, all flat-lay, or all on-location — never mixed within a category. The eye reads inconsistent treatment as "stock images mixed with real product."
4. **Spec table is exact, never approximate.** Per the research §5: "Precise Spec Tables" is checklist item 4. Use exact figures: "Dimensions: 120 × 80 × 45 cm" not "approximately 120cm wide." If a spec is unknown, omit the row — do not write "TBD."
5. **Pricing is either embedded per-SKU or referenced via "Request Quote."** Mixed approach (some SKUs priced, some quote-only) is acceptable but the quote-only SKUs must say "Request Quote" explicitly, not blank. For Indonesian B2B: PPN 11% disclosure on pricing reference page.
6. **Lookbook spreads (if used) interspersed every 4–6 grid pages.** Per the research §5, lookbook provides editorial breathing room. Hard rule: lookbook content uses 3-column system; grid content uses 4-column system. Mixing systems on the same spread fails the grid consistency rule.
7. **Order CTA states MOQ, lead time, shipping zones.** "Order now" is wallpaper. "MOQ 50 units. Lead time 14 days. Ships JABODETABEK same-day, outer Java +3 days, international quote on request." Specific.

## Do-NOT Patterns (3+)

1. **Inconsistent grid usage across catalog.** Switching between 3-col, 4-col, and free-form layout signals lack of design system and confuses navigational scanning. Per research §5 do-not #1.
2. **"Information suffocation" — lack of whitespace in product grid.** Cramming 12+ SKUs per page with no margin between blocks. Per the research §5: catalog grid pages need outer margin minimum 15mm + inter-SKU gutter minimum 8mm. White space is part of the design system.
3. **Navigational failure — missing TOC or unclear category structure.** Per research §5 do-not #3. For 8+ page catalogs, TOC is mandatory. Categories must be visually distinct (chapter dividers, color-coding, or running header).
4. **Mixed product photography treatment within a category.** Half the chairs on white background, half on location — reads as a Frankenstein catalog. Pick one treatment per category.
5. **"TBD" or "approximate" spec entries.** Spec table demands precision. If the spec is unknown, omit the row.
6. **Pricing inconsistency without explicit "Request Quote" framing.** A SKU with no price next to a SKU with a price reads as "the priced one is on sale" or "this catalog is incomplete." If a SKU is quote-only, label it "Request Quote" explicitly.
7. **Cover that doubles as TOC.** Cover is positioning; TOC is navigation. Combining the two cuts both jobs in half.

## Anti-Slop Hard Rules

1. **TOC mandatory for catalogs ≥ 8 pages.** Hard rule. Validate-fail if absent.
2. **Grid consistency across all product pages.** Same column system, same SKU block dimensions, same spec table structure. Hard rule.
3. **Product photography consistent treatment per category.** Hard rule.
4. **Spec tables exact, never approximate.** Hard rule. "TBD" or "approximately" = validate-fail.
5. **Outer margin ≥ 15mm + inter-SKU gutter ≥ 8mm on grid pages.** Print whitespace discipline.
6. **PPN 11% disclosure on pricing page for Indonesian distribution.** Mandatory regulatory.
7. **Lookbook (if used) uses 3-col, grid uses 4-col — no mixing on same spread.** Hard rule.
8. **No banned vocabulary.** Global ban list applies. Additional catalog-product bans: "Premium quality", "Crafted with care", "Exquisite design", "Timeless elegance" — replace with material disclosure (e.g., "Solid oak, FSC-certified") and named designer or origin.
